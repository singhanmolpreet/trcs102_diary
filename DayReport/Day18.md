# Day 18 - Decision Tree Classifier and Confusion Matrix

## Date

Day 18 of ML Training

## Objective

To understand the working of the Decision Tree Classifier algorithm and learn how to evaluate classification models using the Confusion Matrix.

## Topics Covered

### 1. Decision Tree Classifier

* Introduction to Decision Trees
* Supervised Learning for Classification
* Tree Structure:

  * Root Node
  * Internal Nodes
  * Leaf Nodes
* Decision Making using feature splits
* Advantages and limitations of Decision Trees

### 2. Model Training

* Preparing the dataset
* Training a Decision Tree Classifier
* Making predictions on test data

### 3. Confusion Matrix

* What is a Confusion Matrix?
* Components:

  * True Positive (TP)
  * True Negative (TN)
  * False Positive (FP)
  * False Negative (FN)
* Interpreting classification results

### 4. Classification Metrics

* Accuracy
* Precision
* Recall
* F1-Score
* Classification Report

## Activities Performed

* Trained a Decision Tree Classifier on a dataset.
* Predicted class labels using the trained model.
* Generated and interpreted the Confusion Matrix.
* Calculated Accuracy, Precision, Recall, and F1-Score.
* Evaluated the overall performance of the classification model.

## Key Learnings

* Decision Trees classify data by recursively splitting features based on decision rules.
* They are easy to interpret and visualize.
* The Confusion Matrix provides a detailed summary of model predictions.
* Accuracy alone is not always sufficient; Precision, Recall, and F1-Score offer a better understanding of model performance, especially for imbalanced datasets.

## Summary

Day 18 focused on the **Decision Tree Classifier**, a widely used supervised learning algorithm for classification tasks. The session also introduced the **Confusion Matrix** and various classification metrics to evaluate model performance effectively.
